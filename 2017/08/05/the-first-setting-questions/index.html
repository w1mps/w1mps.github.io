记第一次出题

发表于 2017-08-05 | 分类于 ctf

吼吼吼吼，小组的招新已经到了第二轮啦，由于一些事情导致没时间静下来出web题，于是就出了3道misc给大家玩玩，嗨呀~

Hooooooop

音乐响起来的时候有没有暗道一声“卧槽”，hhhhh。前面的音频是正常的，可是后面又不对劲的地方出现。用Au打开音频文件，在显示频谱的地方可以看到，

可能有的盆友会被百度上的ctf题目误导是嘟嘟嘟嘟嘟的摩斯密码，其实不然，这里放大频谱图，把黑色的地方画下来，很明显的猪圈密码。知道就很简单，不知道就会觉得很难，这也是个人积累的问题，多刷题多刷题多刷题，然后对应解密就好了。

flag:D0g3{ahhhhhhhhh}

a shhhhhhhh~

题目到手，binwalk跑一下，发现藏了zip，而且压了两张图片在里面，用dd将zip文件分割出来或者直接用16进制编辑器打开手动取出来，再尝试解压，需要解压密码，又根据hint提示了logo.png，猜测明文攻击。题目描述里提到了关键的东西在附近，于是想到回到首页，查看源码，发现了logo.png。
这里推荐一下AZPR这个工具，功能比较全面，这里只需将下载下来的logo.png压缩，但是要注意压缩算法。
于是打开工具：


得到压缩密码lalalala，解压出来了一张hidden.jpg。emmmm，jpg啊。stegdetect看一下，可以知道是jphide，然后对应下载工具利用即可。至于原理，有兴趣可以了解一下，不得不说图片里的学问也很多。贴上相关链接：
http://www.cnblogs.com/daban/p/5680615.html
利用工具解出来一个txt，base64解码两次就是flag。

flag:D0g3{jph1de_4_fun}

The emoji system

下载得到一个png文件，一般png文件可以用stegsolve看看，也许有惊喜，这里变换通道发现：

得到了个key和LSB提示，这里本来想的是只放一个通道的，结果害怕你们看不到，就直接RGB都放了，这里贴出一篇参考文章，于是用key解出来一个文件（本来想把key放到其他地方的，但是想着简单点就一起给了，没想到会被当成flag提交），winhex打开看到IHDR，知道是png图片然后补上头，看到这个二维码是经过处理的，还需要用修复一下。这里有个小坑的地方就是我把右下角的一块反相了【2333333，修复完成，扫码即可。
